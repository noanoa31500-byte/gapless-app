## 🌊 なぜこの機能が、泥水で視界が悪い洪水時に有効なのか

### 1. 感電死は「見えない死」

タイの洪水では、**水没した電柱・電線からの漏電による感電死**が深刻な問題です。

#### 問題の本質
- 濁った水の中では、沈んだ電線を目視で確認することは**不可能**
- 水は電気を通すため、電線に触れなくても周囲の水に電流が流れる
- 115kVの高圧送電線が水没すると、半径20m以上が危険区域となりうる

#### 本機能の解決策
```
OSMデータから抽出した電力インフラの位置情報を活用
↓
現在地から各電力設備への「方位角」を計算
↓
危険な方向をレーダー上に「黄色のゾーン + ⚡マーク」で表示
↓
ユーザーは「黄色い方向を避ける」だけで感電リスクを回避
```

### 2. 激流は突然来る

洪水時、川や排水溝からの水流は予測困難です。

#### 危険な閾値
- **膝上（0.5m以上）の水深**: 大人でも流される危険
- **流速が「Fast」または「Moderate」**: 歩行困難

#### 本機能の解決策
```
浸水シミュレーションデータ（satun_flood_prediction.json）を活用
↓
pred_depth >= 0.5m または pred_speed == "Fast"/"Moderate" を検出
↓
「深くなる方向」「流れが速い方向」を青・紫のゾーンで警告
↓
ユーザーは「色付きの方向を避けて」安全な浅瀬を進む
```

### 3. パニック時の認知負荷軽減

災害時、人は複雑な判断ができません。

#### 認知科学的背景
- ストレス下では前頭前皮質の機能が低下
- 地図を読む（空間認知）は高次の認知処理を要する
- パニック時に「この道は危険か？安全か？」を判断する余裕はない

#### 本機能の解決策
```
複雑な判断 → シンプルな色判断に変換

🟡 黄色 = 感電危険（⚡）→ 絶対に行かない
🔵 青色 = 深水危険（🌊）→ 行かない
🟣 紫色 = 激流危険（💨）→ 行かない  
🟢 緑色 = 安全方向（✅）→ この方向へ進む
```

### 4. 360度全方位の危険を一目で把握

Google Mapsなどの地図アプリでは「前方」しか見えません。

#### レーダー表示の優位性
- **背後からの激流**も同時に把握できる
- 「どの方向に逃げるか」ではなく「どの方向は絶対ダメか」を先に表示
- 視野角360度をカバーする唯一のUI

### アルゴリズム詳細

#### 1. 危険方向の算出（Danger Bearing）

```dart
// 現在地から各危険物への方位角を計算
double _calculateBearing(LatLng from, LatLng to) {
  final lat1 = from.latitude * π / 180;
  final lat2 = to.latitude * π / 180;
  final dLon = (to.longitude - from.longitude) * π / 180;
  
  final y = sin(dLon) * cos(lat2);
  final x = cos(lat1) * sin(lat2) - sin(lat1) * cos(lat2) * cos(dLon);
  
  return (atan2(y, x) * 180 / π + 360) % 360;
}

// 危険物の距離に応じて角度範囲を計算
// 近いほど広い範囲を危険と判定
final angularWidth = 30.0 * (1.0 + severity * 0.5);
```

#### 2. 安全ルートへの誘導（Safety Guidance）

```dart
// ターゲット方向にリスクがある場合、左右に迂回角度を探索
for (double offset = 10; offset <= 90; offset += 10) {
  // 右方向をチェック
  final rightBearing = (targetBearing + offset) % 360;
  if (isSafe(rightBearing)) {
    return correctionAngle = offset; // 右へ迂回
  }
  
  // 左方向をチェック  
  final leftBearing = (targetBearing - offset + 360) % 360;
  if (isSafe(leftBearing)) {
    return correctionAngle = -offset; // 左へ迂回
  }
}
```

## 📊 データソース

### power_risk_th.geojson
- **ソース**: OpenStreetMap Overpass API
- **内容**: タイ・サトゥン県の電力インフラ
  - 送電線（`power=line`）
  - 鉄塔（`power=tower`）
  - 変圧器（`power=transformer`）
- **電圧情報**: 115,000V等（高圧送電線）

### satun_flood_prediction.json
- **内容**: 浸水シミュレーション予測データ
- **フィールド**:
  - `pred_depth`: 予測水深（メートル）
  - `pred_speed`: 予測流速（None/Slow/Moderate/Fast）
  - `risk_score`: リスクスコア
  - `elevation`: 標高

### roads_th.geojson
- **ソース**: OpenStreetMap
- **内容**: 道路ネットワーク
- **用途**: 安全ルートの探索基盤

## 🔒 オフライン動作

本機能は**完全オフライン**で動作します。

### オフラインが重要な理由
1. 災害時は通信インフラが破壊されることが多い
2. 電波塔が水没・倒壊する可能性
3. 回線混雑で接続できない状況

### 実現方法
- すべてのリスクデータはアプリにバンドル
- GPSは衛星直接通信のため通信不要
- 計算はすべて端末内で完結

# ルート追従型スマートコンパス 実装ガイド

## 📍 概要

最大安全ルート上の次のウェイポイントを常に指し示す、災害対応型のスマートコンパスです。

## 🎯 主要機能

### 1. 自動ウェイポイント特定
- ルート上で現在地に最も近い点を特定
- 次の「重要な地点」（曲がり角または10m先）を自動選択
- 進行方向の角度変化を検出（30度以上で曲がり角判定）

### 2. 動的方位計算
- `geolocator`で現在地とターゲット間のBearing（方位）を計算
- `flutter_compass`でデバイスの向きを取得
- 相対角度を計算してコンパスの針を回転

### 3. オフコース判定
- ルートから20m以上離れたら自動検知
- コンパスの色が赤に変化
- バイブレーション警告（TODO）

### 4. 距離表示
- 次のウェイポイントまでの距離（メートル）
- 目的地までの総距離（キロメートル）

## 🚀 使用方法

### 基本的な使い方

```dart
// 1. ウィジェットを配置
SmartCompassWidget(
  size: 200.0,
)

// 2. ルートが計算されると自動的に追従開始
// （ShelterProvider.safestRouteを監視）
```

### パフォーマンス最適化のポイント

1. **StreamBuilder使用**
   - センサーデータを効率的に処理
   - 不要な再描画を防止

2. **アニメーション最適化**
   - 300msの滑らかな遷移
   - 針の急な動きを抑制

3. **計算の最適化**
   - ウェイポイント更新は現在地変更時のみ
   - 重い計算をStreamで非同期化

## 📐 技術的詳細

### 方位計算ロジック

```dart
// 1. ターゲット方位を計算（0-360度）
final targetBearing = Geolocator.bearingBetween(
  currentLat, currentLng,
  targetLat, targetLng,
);

// 2. デバイスの向きとの差分を計算
double rotation = targetBearing - deviceHeading;

// 3. -180～180度に正規化
while (rotation > 180) rotation -= 360;
while (rotation < -180) rotation += 360;
```

### ウェイポイント選択アルゴリズム

```
1. 現在地から最寄りのルート上の点を検索
   ↓
2. その点から10m以上先を探索
   ↓
3. さらに先（5点以内）に30度以上の曲がり角があるか確認
   ↓
4. あれば曲がり角の手前、なければ10m先の点を選択
```

## ⚠️ オフライン環境での注意点

### 地磁気偏角の補正

日本とタイでは地磁気偏角が異なります：
- **日本（東京）**: 約-7度（西偏）
- **タイ（バンコク）**: 約0度

現在の実装では`geolocator`が自動的に真方位を計算するため、
明示的な補正は不要です。

### センサー精度

- **flutter_compass**: 地磁気センサーの精度に依存
- **周囲の磁気干渉**: 鉄筋建物内では精度が低下
- **推奨**: 屋外での使用

### バッテリー消費対策

1. 使用していない時はStreamを停止
2. 更新頻度を適切に制限（現在300ms）
3. バックグラウンドでの自動停止機能

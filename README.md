## 🎯 プロジェクトビジョン

**「災害時、判断力を失った人間を、システムが導く」**

従来の防災アプリは「情報提供」にとどまります。Gaplessは、パニック状態のユーザーを**物理的に避難所まで導く**ことに特化した、次世代の災害対応システムです。

---

## ⚡ なぜこれが革新的なのか

### 1. 🧠 Offline AI Triage（オフライン判断システム）
- **課題:** 災害時、通信インフラは最初に途絶する
- **解決:** 端末内で怪我の重症度を判断し、適切な避難所（医療設備の有無）へルート変更
- **技術:** ローカルAIモデル + ルールベース判断システム

### 2. 📳 Panic UI（パニックモード）
- **課題:** 災害時、画面を見る余裕はない
- **解決:** ポケットに入れたまま、「振動パターン」と「音声」だけで避難可能
- **技術:** 
  - **Native:** 振動パターンによる触覚フィードバック
  - **Web:** 画面フラッシュによる視覚的警告
  - **Voice:** 「あと300m、北東」と音声で誘導

### 3. 🚨 Global Disaster Interrupt（強制割り込み）
- **課題:** 平時の画面（設定やチャット）にいても、緊急地震速報などでパニックになると操作できない
- **解決:** 災害モードが発動（またはスイッチON）した瞬間、**アプリ内のどこにいても全履歴を消去してコンパス画面へ強制移動**
- **実装:** `main.dart` レベルでの常時監視システム (`DisasterWatcher`)

### 4. 🧭 Reactive Navigation（即応型コンパス）
- **課題:** 目的地を変更したくても、再検索や設定が面倒
- **解決:** 
  - **Chat-to-Nav:** AIチャットで「ここに行く」を押すだけで、即座に目的地ロック＆ナビ開始
  - **Reactive UI:** 目的地が変わった瞬間、コンパスの針がギュンと動き、新しい方角を指し示す
  - **Auto Zoom:** 起動時に現在地と最寄り避難所が収まるよう自動ズーム

### 5. 🌏 Hyper Localization（超ローカライゼーション）
- **課題:** 海外展開時、各国のハザードマップを手作業で統合するのは非現実的
- **解決:** GeoJSON形式のハザードデータを動的読み込み、地域ごとに切り替え
- **実装地域:**
  - 🇯🇵 **宮城県大崎市**: 日本の地方都市モデル
  - 🇹🇭 **サトゥン県**: タイの洪水多発地域
- **言語:** 日本語・英語・タイ語（緊急時用の短文最適化）

## 🛠️ 技術スタック

### Frontend
- **Flutter (Dart)**: クロスプラットフォーム対応（iOS/Android/Web）
- **Provider**: 状態管理（全画面同期）
- **flutter_map**: OpenStreetMap統合
- **flutter_tts**: 音声ガイダンス
- **vibration**: ハプティックフィードバック

### Data & Backend
- **Python**: ハザードデータ解析（GeoJSON処理）
- **JSON**: 避難所・ハザードマップデータ管理

### Platform Features
- **位置情報 (GPS)**: geolocator（自動起動リクエスト）
- **コンパス**: flutter_compass
- **Web対応**: kIsWeb分岐によるプラットフォーム固有機能

## 🚀 デモ体験

その後、別ターミナルで：

## 🎮 隠しデモ機能

審査員向けに、3つの地点へ瞬時に移動できる「テレポート機能」を実装しています。

**使い方:**
1. 設定画面を開く
2. 画面下部の「バージョン情報」を**長押し**
3. Developer Menuが表示される
4. 好きな場所にテレポートすると、周囲の避難所やハザートマップも自動で切り替わります

**テレポート先:**
- 🏫 **School (Here)**: デモ会場（リアルタイム実演用）
- 🇯🇵 **Osaki City Hall**: 大崎市役所（シナリオデモ用）
- 🇹🇭 **PCSHS Thailand**: タイの学校（国際展開デモ用）
